# 🛒 FreshMart - Premium Groceries Platform

FreshMart is a complete, full-stack web application designed for a premium grocery delivery business. It is separated into three distinct codebases: a customer-facing frontend, an admin management dashboard, and a centralized Node.js/Supabase backend to power them both.

Recently, the application was upgraded to include a **Custom Local AI Image Recognition** system built entirely from scratch to help customers find products by just uploading a photo.

---

## 📂 Project Structure

The project is housed inside the `e:\Arul\freshmart` directory and is split into three main folders:

### 1. `backend/` (The Server)
The API layer connecting the application to the database and handling business logic.
- **Tech Stack:** Node.js, Express.js, Supabase (PostgreSQL), TensorFlow.js.
- **Key Directories & Files:**
  - `src/index.js`: The main entry point initializing the Express server (usually runs on port `3001`).
  - `src/supabase.js`: Configuration to connect to the Supabase PostgreSQL database.
  - `src/routes/`: Contains all the API endpoints (`products.js`, `orders.js`, `flash-deals.js`, `ai.js`, etc.).
  - `scripts/`: Custom machine learning scripts (`collect_dataset.js`, `train_model.js`) that automatically pull product pictures from the database and train the AI.
  - `models/freshmart_model/`: Stores the custom convolutional neural network (CNN) `.json` weights trained for local image recognition.

### 2. `frontend/` (The Customer UI)
The storefront where customers browse products, manage their cart, and use advanced features.
- **Tech Stack:** React (Vite), React Router, Lucide-React for icons.
- **Key Directories & Files:**
  - `src/pages/`: Main application views like `Home.jsx`, `Products.jsx`, `Checkout.jsx`.
  - `src/components/`: Reusable interface components like `Header.jsx`, `ProductCard.jsx`, `LocationSidebar.jsx`. 
  - `src/styles/global.css`: A centralized, tailored CSS file enforcing a premium, glassy, and modern aesthetic.
  - **Feature Highlights:** Live flash deals synced to the database, cart logic management via Context API, and an AI Image Search feature built right into the search bar.

### 3. `admin/` (The Management UI)
The internal portal for store managers to oversee operations.
- **Tech Stack:** React (Vite).
- **Key Directories & Files:**
  - `src/pages/`: Dashboard views like `Inventory.jsx` to restock items, `Orders.jsx` to update delivery statuses, and `FlashDeals.jsx` to configure live homepage sales.
  - `src/api.js`: Axios configurations pointing to the backend to perform protected modifications.
  - `src/styles/admin.css`: Professional dashboard styling tailored for tabular data and metric tracking.

## 🚀 How to Run the Project

To successfully boot up the entire FreshMart ecosystem, you need to run three separate terminal processes, one taking place inside each main folder.

### 1. Start the Backend API
```bash
cd backend
npm install
npm run dev      # Used if you want automatic restarts when modifying code
# OR
npm start        # Used for standard startup
```
*The backend API will be available at `http://localhost:3001`.*

### 2. Start the Customer Frontend
```bash
cd frontend
npm install
npm run dev
```
*The customer site will usually be available at `http://localhost:5173`.*

### 3. Start the Admin Panel
```bash
cd admin
npm install
npm run dev
```
*The admin panel will usually be available at `http://localhost:5174`.*

## 🤖 The Custom AI Image Recognition System

One of the application's core features is the locally hosted AI model meant to help customers find products without typing. 

1. **How it works:** A customer clicks the camera icon in the `Header.jsx` search bar and uploads an image (e.g. an apple). The image is converted into bits and sent to `/api/ai/recognize-image`.
2. **Local Inference:** Rather than relying on external paid APIs like OpenAI or Gemini, the Node server directly feeds the image picture through a Custom TensorFlow Neural Network generated by `train_model.js`.
3. **The Result:** The model outputs mathematical probabilities. The server finds the highest match (e.g. "98% confident this is a Shimla Apple") and tells the frontend, which automatically displays the item to the user!
